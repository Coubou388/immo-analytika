1

Installer Git pour Windows

Le seul logiciel à installer — rapide et gratuit

Une seule fois
a

Va sur git-scm.com/download/win et télécharge "64-bit Git for Windows Setup".

b

Lance l'installeur. Garde toutes les options par défaut et clique "Next" jusqu'à la fin. Clique "Install".

c

Vérifie l'installation : ouvre PowerShell (touche Windows → tape "powershell" → Entrée) et tape :

PowerShell
git --version
✓ Si tu vois "git version 2.x.x", c'est bon.
d

Configure ton identité Git (remplace avec ton vrai nom et email GitHub) :

git config --global user.name "Coubou"
git config --global user.email "COUBOUDIABAGATE@gmail.com"
2

Créer le repo GitHub et y uploader les fichiers

Le repo sera la source de vérité — Netlify le lit automatiquement

Une seule fois
a

Va sur github.com/new. Crée un repo avec ces paramètres :

Nom : immo-analytika
Visibilité : Public ✓
☑ Add a README file

Clique "Create repository".

b

Note l'URL de ton repo — elle ressemble à :

https://github.com/TON-USERNAME/immo-analytika
c

Ouvre PowerShell. Trouve d'abord le chemin Windows exact de ton dossier Engagement. Dans l'Explorateur Windows, navigue vers :

Immo+ 2025 → 02 Projects → Immo+ Engagement

Clique sur la barre d'adresse en haut → copie le chemin complet qui apparaît.

Exemple : C:\Users\Coubou\Documents\Immo+ 2025\02 Projects\Immo+ Engagement
d

Dans PowerShell, initialise git dans ce dossier (remplace le chemin) :

cd "C:\Users\Coubou\Documents\Immo+ 2025\02 Projects\Immo+ Engagement"
git init
git add *.html
git commit -m "Initial deploy — rapports Immo+"
git branch -M main
git remote add origin https://github.com/TON-USERNAME/immo-analytika.git
git push -u origin main
⚠️ Git va te demander de te connecter à GitHub. Une fenêtre de navigateur s'ouvrira — connecte-toi avec ton compte GitHub. C'est uniquement la première fois.
e

Vérifie sur github.com/TON-USERNAME/immo-analytika que tes fichiers HTML sont bien là.

3

Connecter Netlify à GitHub

Netlify redéploie automatiquement à chaque push GitHub

Une seule fois
a

Va sur netlify.com → "Sign up" → connecte-toi avec ton compte GitHub. C'est gratuit.

b

Dans le dashboard Netlify, clique "Add new site" → "Import an existing project".

c

Choisis GitHub → autorise Netlify → sélectionne le repo immo-analytika.

d

Sur la page de configuration, laisse tout par défaut et clique "Deploy site". Netlify déploie en 1–2 minutes.

💡
Optionnel : personnalise l'URL

Dans Site settings → General → Site name, tu peux changer le nom aléatoire en quelque chose comme immo-analytika pour avoir l'URL immo-analytika.netlify.app.

e

Note ton URL Netlify finale — tu en auras besoin pour configurer le script PowerShell à l'étape suivante.

https://immo-analytika.netlify.app
4

Configurer le script PowerShell Auto-Deploy

Le script est déjà créé par Claude — juste 2 lignes à modifier

⚠️ Requis
a

Dans ton dossier Immo+ Engagement, trouve le fichier [C] Auto-Deploy — Watcher.ps1. Fais un clic droit → "Modifier avec le Bloc-notes".

b

Remplace les deux lignes suivantes avec tes vraies valeurs :

# Ligne 16 — remplace CHEMIN_DU_REPO par ton vrai chemin
$repoPath = "C:\Users\Coubou\Documents\Immo+ 2025\02 Projects\Immo+ Engagement"

# Ligne 17 — remplace par ton URL Netlify
$netlifyUrl = "immo-analytika.netlify.app"

Sauvegarde avec Ctrl+S et ferme le Bloc-notes.

c

Autorise PowerShell à exécuter des scripts locaux (une seule fois) :

# Lance PowerShell en tant qu'Administrateur, puis tape :
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
# Tape Y et Entrée pour confirmer
5

Lancer le Watcher — pipeline actif

À faire au début de chaque session de travail avec Claude

a

Ouvre PowerShell. Navigue vers ton dossier et lance le script :

cd "C:\Users\Coubou\...\Immo+ Engagement"
.\[C] Auto-Deploy — Watcher.ps1
b

Tu verras ce message confirmant que le watcher est actif :

================================================
Immo+ Auto-Deploy Watcher — ACTIF
================================================
Dossier surveille : C:\...\Immo+ Engagement
Site en ligne : https://immo-analytika.netlify.app
En attente de modifications...
c

Laisse cette fenêtre PowerShell ouverte pendant que tu travailles avec Claude. Chaque modification d'un fichier .html déclenchera automatiquement un push vers GitHub et un redéploiement Netlify.

✅
Délai total : environ 1 à 2 minutes

Claude sauve → PowerShell détecte (3–8 sec) → Git push (5–10 sec) → Netlify redéploie (30–90 sec) → Page mise à jour.

d

Pour arrêter le watcher : clique dans la fenêtre PowerShell et appuie sur Ctrl+C.

🎉

Pipeline opérationnel !

À partir de maintenant, chaque fois que Claude produit ou met à jour un rapport HTML, il sera automatiquement en ligne en moins de 2 minutes. Zéro action manuelle de ta part.
